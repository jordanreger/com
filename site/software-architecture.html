software architecture
software is not art but architecture.

software is used to design and build useful things, just like architecture.

architecture is pragmatic art.

therefore, software is pragmatic art.


why make this distinction?

because art is made for art's sake - software never is.

art can be made with software, but software itself is not art - it's architecture.

why can't software be art?

art can be dysfunctional and it stays art.

architecture cannot be dysfunctional and be architecture still.

it is a failure.

software is the same.


software is architecture.
